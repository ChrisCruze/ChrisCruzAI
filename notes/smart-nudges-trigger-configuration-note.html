Trigger Design

Trigger Configuration Matrix

Updated September 15, 2025

Mission

Give product and lifecycle teams a declarative toolkit to orchestrate user, event, and time-based nudges with confidence, while preserving compliance guardrails and measurement fidelity.

Matrix Dimensions

  • • User: Lifecycle stage, attendance streak, reciprocity score, host/vip tags.
  • • Event: Capacity, category, host responsiveness, waitlist depth.
  • • Time: Countdown windows, weekday & hour heuristics, quiet hours.

Guardrails

  • • Frequency caps at user + channel level with override approvals logged.
  • • Compliance flags for scarcity claims validated via Control Tower data.
  • • Suppression lists for churned or paused members synced hourly.
  • • Regression tests before publishing new YAML recipes.

Sample YAML Snippet

recipe: scarcity_last_call
user:
  stage: active
  reciprocity_score: ">=3"
event:
  capacity_remaining: "<=3"
  host_trust: "high"
time:
  window_before_start: "PT12H"
channel: sms
provider: twilio
copy_template: scarcity_last_call_v2
experiments:
  - name: proof_variant
    personalization: attendee_social_handles
    holdout: 10%

Operational Cadence

Weekly

Prioritize new recipes based on insights from the Control Tower Blueprint.

Daily

Review failed runs and suppression reasons in the Delivery Guide.

Release

Peer-review YAML changes, run automated regression tests, and capture diff notes in Git.

Monthly

Audit reciprocity-driven nudges with the Psychology Evidence Pack to ensure alignment.

Operator Moves

  • • Version control every YAML file and link pull requests to dashboard charts showing expected impact.
  • • Partner with data science to update heuristics when seat sell-through or RSVP curves shift.
  • • Surface new trigger ideas in the Four-Pillar Strategy meeting for prioritization.
